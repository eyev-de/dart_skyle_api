Methods

connect() → Future<void>
Starts the connection process of Skyle. First the ethernet interfaces are scanned for the right IP address. If the correct IP address is found, the grpc client is created. One API call is tested until the grpc library does not throw an exception anymore. This function blocks until Skyle is connected but it can also be called and not awaited which results in an async re/connection cycle.
disconnect() → Future<void>
Disconnects Skyle and shuts down the reconnection cycle.
dispose() → void
Disposes the instance of ET completely.
